K-Agile Guide

The Operational Framework for the AI Era

Official Version 1.0 • March 24, 2026

1. Purpose of the Guide

K-Agile is an operational framework for developing and maintaining complex products, heavily inspired by Agile. This guide defines the roles, events, artifacts, and rules that bind these elements together. K-Agile is not a linear methodology but a circular ecosystem of knowledge. The "K" stands for Knowledge. It is in no way an alternative to Agile; rather, it embraces and enhances all its original principles.

2. Definition

K-Agile is based on the awareness that Artificial Intelligence is not just a tool, but a cognitive agent that requires a structured context to function.

2.1 The Three Pillars of Augmented Empiricism

  • 1. Radical Transparency: Every decision (meeting/code) must be visible to the AI to prevent hallucinations. The entire codebase must be made known to the AI, which must demonstrate an understanding of its historical context, dependencies, style, and standards.
  • 2. Semantic Inspection: The team doesn't just check if the code "runs," but whether the meaning (the intent) has been respected.
  • 3. Prompt Adaptation: Continuous improvement involves refining the language and context provided to the machine.

2.2 K-Agile Values

  • Courage: To discard fast-generated code if it is not understood.
  • Focus: On business intent, leaving syntax to the machine.
  • Responsibility: The "Context Owner" is the guarantor of the system's truth.
  • Explainability: The ability to explain AI-driven choices to regulators or users.

3. The K-Agile Team

3.1 The Product Owner

The Visionary Prototyper

The PO is the first link in the technical production chain. They use AI to create "functional drafts" (MVPs). If a Story cannot be visualized or simulated by AI, it is not "Ready". This is mandatory after requirements gathering.

3.2 The Developer

The Context Owner & Master Prompter

The Developer is the director of software construction. They no longer write code; they orchestrate AI.

The RAG Methodology (Retrieval-Augmented Generation)

Retrieval: The system queries the K-Base for code snippets and meeting transcripts.

Augmentation: Info is injected into the prompt for a contextualized instruction.

Generation: AI generates the final response based exclusively on provided data, eliminating hallucinations.

3.3 The Scrum Master

Flow & Governance Architect

The SM facilitates Human-AI synergy and guarantees system integrity. They monitor for "AI Drift" and cognitive burnout.

  • Compliance: Vigilance over AI Act, GDPR, and IP protection.
  • Anti-Bias: Ensures algorithms don't introduce technical or logical prejudices.
  • Audit: If the system becomes a "black box", the SM halts the sprint.

4. K-Agile Events

"AI speed must never exceed the team's capacity for understanding. The one-week Sprint is the anchor."

The 1-week Sprint is the optimal choice to balance generative acceleration with human governance.

Neural Briefing

Converts verbal conversations into Context Packages (goals and prompt-ready intent).

Augmented Daily

Analysis of the "Context Delta": what changed in the last 24h? Is the AI aligned?

Context Refactoring

Hygiene ceremony to prevent context degradation and remove obsolete prompts.

6. Definition of Done (DoD)

  1. 1. Code generated or validated through Prompting.
  2. 2. K-Base updated and re-indexed by the AI.
  3. 3. AI can explain the feature's logic (Explainability).
  4. 4. Product complies with AI Ethics values and regulations.
